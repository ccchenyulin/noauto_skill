````markdown
| `OUTPUT_NODE` | `bool` | 设为 `True` 表示该节点是终端输出节点（如保存图片），ComfyUI 会强制执行它。默认不设置（等同于 False）。 |

标准模板：

```python
class NodeName:
    @classmethod
    def INPUT_TYPES(cls):
        return {
            "required": {},
            "optional": {}
        }

    RETURN_TYPES = ()   # 输出类型元组，例如 ("IMAGE", "MASK")
    RETURN_NAMES = ()   # 输出名称元组，例如 ("输出图像", "输出遮罩")
    FUNCTION = "run"    # 节点执行时调用的方法名
    CATEGORY = ""       # 节点在 ComfyUI 菜单中的分类路径
    # OUTPUT_NODE = True  # 如果是终端输出节点（如保存文件），取消此行注释

        pass
```

#### 日志分级规范

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| `DEBUG` | 变量值、张量形状、中间计算结果、循环进度 | 每个张量操作前后的 shape |

#### 必须打日志的位置（全部必填，不得遗漏）

1. **方法入口**：打印所有输入参数的值和张量 shape
2. **每个条件分支入口**：说明进入了哪个分支，以及触发条件的值

4. **每次循环的关键步骤**：打印当前迭代索引和关键中间值
5. **调用外部函数/库前后**：打印传入参数和返回结果的关键信息
6. **每个 try 块入口**：说明即将尝试什么操作
7. **每个 except 块**：打印异常类型、异常信息、完整堆栈
8. **方法出口**：打印所有输出的关键信息（shape、值范围等）

#### 日志示例（标准）

```python
def run(self, image, threshold, mode):
    # ── 入口日志 ──────────────────────────────────────
    logger.info(f"[MyNode][INFO] ══ 节点开始执行 ══")
    logger.debug(f"[MyNode][DEBUG] 输入参数：")
    logger.debug(f"[MyNode][DEBUG]   image.shape  = {image.shape}")
    logger.debug(f"[MyNode][DEBUG]   image.dtype  = {image.dtype}")
    logger.debug(f"[MyNode][DEBUG]   image.device = {image.device}")
    logger.debug(f"[MyNode][DEBUG]   threshold    = {threshold}")
    logger.debug(f"[MyNode][DEBUG]   mode         = {mode}")

    # ── 输入合法性检查 ─────────────────────────────────
    logger.info(f"[MyNode][INFO] 开始输入合法性检查...")
    if image.ndim != 4:
        logger.error(f"[MyNode][ERROR] 输入图像维度错误，期望 4 维，实际 {image.ndim} 维")
        raise ValueError(f"[MyNode] 输入图像必须是 4 维张量 [B,H,W,C]，当前维度为 {image.ndim}")
    logger.info(f"[MyNode][INFO] 输入合法性检查通过")

    try:
        # ── 分支判断 ───────────────────────────────────
        logger.info(f"[MyNode][INFO] 判断执行模式，当前 mode={mode}")
        if mode == "fast":
            logger.info(f"[MyNode][INFO] 进入快速模式分支")
            # 具体处理逻辑...
            logger.debug(f"[MyNode][DEBUG] 快速模式处理完成，中间张量 shape={image.shape}")
        else:
            logger.info(f"[MyNode][INFO] 进入精细模式分支")
            # 具体处理逻辑...

        # ── 张量操作日志 ───────────────────────────────
        logger.debug(f"[MyNode][DEBUG] 即将执行阈值过滤，threshold={threshold}")
        result = image * threshold
        logger.debug(f"[MyNode][DEBUG] 阈值过滤完成，result.shape={result.shape}，值范围=[{result.min():.4f}, {result.max():.4f}]")

        # ── 出口日志 ───────────────────────────────────
        logger.info(f"[MyNode][INFO] ══ 节点执行完成 ══")
        logger.debug(f"[MyNode][DEBUG] 输出结果：result.shape={result.shape}")
        return (result,)

    except ValueError as e:
        logger.error(f"[MyNode][ERROR] 数值错误：{e}", exc_info=True)
        raise
    except Exception as e:
        logger.error(f"[MyNode][ERROR] 未知错误：{e}", exc_info=True)
        raise
```

缺失任意位置的日志视为代码不合格，必须补全后才能提交。

### 7. 错误处理规范

1. **捕获具体异常优先**：能预判的错误先单独捕获并给出中文友好提示
2. **兜底捕获 Exception**：在最外层用 `except Exception as e` 兜底，记录日志后 `raise` 重新抛出
3. **禁止返回 None 作为图像**：出错时直接 `raise`，不返回 `(None,)` 让后续节点崩溃
4. **入口合法性检查**：在方法入口对关键输入做基本校验，不合法时抛出带中文说明的 `ValueError`

### 输出格式（固定，必须严格遵守）

使用以下固定标记包裹（方便脚本提取）：

```
%%LOGIC_START%%
【节点名称】XXXNode 执行逻辑

【第 1 步】步骤名称
  - 动作：具体在做什么（用大白话，不要出现代码）
  - 输入：这一步接收什么数据，数据长什么样
  - 输出：这一步产出什么数据，数据变成什么样
  - 可能出错：这一步可能遇到什么问题

【第 2 步】步骤名称
  - 动作：...
  - 输入：...
  - 输出：...
  - 可能出错：...

（以此类推，每个独立操作都是一步，不得合并）

【数据流向图】
输入参数A ──→ [步骤1: 做什么] ──→ 中间结果X
输入参数B ──→ [步骤2: 做什么] ──→ 中间结果Y

## 规则

必须遵守：

1. 节点必须完整可运行
2. 代码必须符合 ComfyUI API
3. 不破坏 user_mwh 结构
4. 参数必须有 `tooltip` 中文说明
5. 输出代码必须完整
6. 每轮回复末尾必须包含对话控制中定义的询问语句
7. 仅当用户明确回复"完成"时，才执行文件更新步骤
8. 每轮回复必须包含进度快照（在询问之前）
9. 进度快照自动继承上一轮的值，无需用户同意
10. 日志必须极尽详细，覆盖所有规定位置，缺失任意位置视为未完成
11. 注释必须详细、中文，面向 Python 初学者
12. 有第三方依赖时必须在文件顶部和节点说明中声明安装命令
13. `__init__.py` 更新必须使用增量 patch 格式，不得输出整个文件
14. 工作流连接示例和验证方法为必填项，不可省略
15. 所有代码、逻辑描述、进度快照必须使用规定的 `%%标记%%` 包裹
16. 修复已有节点时必须先要求用户提供原始代码，严禁臆测
17. 修复流程必须分两阶段执行：第一阶段只加日志，第二阶段才修逻辑
18. 每次改动必须遵守最小改动原则，不得顺带修改无关代码
19. 每次改动必须在进度快照的"本轮变更记录"字段中注明改动内容
20. **收到提示词后的第一次响应只能输出就绪回复，严禁直接开始开发或输出代码**
21. 用户说"交接"时必须立即生成完整交接文档，不得省略任何字段
22. 收到交接文档时必须先输出理解确认回复，确认无误后再等待用户指示下一步动作
````
